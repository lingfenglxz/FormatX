# 编辑器与结构树可用性优化

## 范围

- 左侧 JSON 编辑器粘贴或输入后，不得出现 `tok-key` 等高亮层 HTML。
- 搜索选中文本时，文字必须保持可读。
- 右侧树的可展开箭头统一置于节点键名左侧；根节点保持现有位置。
- 每个树层级缩进固定为 `2ch`。
- Windows 输出 NSIS、MSI 与便携 ZIP 三种交付物；MSI 显式采用简体中文安装界面。

## 方案

编辑器继续使用现有 `textarea + 只读高亮镜像`，但让镜像只负责绘制、编辑层只保留原始文本，避免 DOM 高亮标记进入文档内容。选区使用固定前景色而不使用混色模式。

树继续使用现有递归组件。将每个属性的键名和子节点标题合并为一行，使子节点的箭头先于键名渲染；子内容容器的左侧缩进改为 `2ch`。不引入新树控件或依赖。

MSI 通过 Tauri/WiX 的语言配置指定 `zh-CN`。NSIS 是传统轻量 EXE 安装包；MSI 是 Windows Installer 标准包，便于企业分发、静默安装和系统统一管理。便携版复用同一次 Tauri 构建生成的 `FormatX.exe`，用项目脚本打包为 ZIP；解压后可直接运行，不注册卸载项、不改 PATH。

## 验证

- 单测覆盖：JSON 高亮结果不含高亮标签作为文本、树节点标签/箭头顺序，以及编辑器选择样式。
- 运行前端测试与生产构建。
- 构建三种 Windows 交付物，检查 ZIP 内只含可直接运行的 `FormatX.exe`，并确认 MSI 的语言配置被纳入构建输入。
