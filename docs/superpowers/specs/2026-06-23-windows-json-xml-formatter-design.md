# FormatX — 项目设计文档

## 1. 项目目标

制作一款名为 **FormatX** 的离线 Windows 桌面小工具，用于格式化、压缩和查看 JSON、XML 文本。用户在左侧编辑器粘贴或修改内容，格式化或压缩后直接替换该编辑器中的原文；右侧以树状结构展示同一份数据，并允许编辑后实时同步回左侧。

首版仅支持 JSON 与 XML，不增加 YAML、SQL、文件管理、多标签页或云端同步等功能。

## 2. 技术选型

- 桌面壳：Tauri 2
- 前端：React + TypeScript
- 构建产物：Windows 安装包与便携式可执行文件
- 数据处理：完全本地执行；不上传、不保存用户粘贴内容

选择 Tauri 而不是 Electron，原因是本工具只需要单窗口文本处理，Tauri 的安装包和运行开销更小。

## 3. 页面与交互

### 3.1 总体布局

单窗口开发工具风格，主体左右分栏：

- 顶部工具栏：FormatX 标识、格式类型切换（JSON / XML）、格式化、压缩、复制、清空、查找、主题切换。
- 左侧：唯一的文本编辑器。粘贴、输入、格式化和压缩均发生在此处；处理结果原地替换文本。
- 右侧：紧凑可编辑树。取消原型中无实际作用的 `TREE VIEW` 标题、重复输出区、多标签页及独立状态。
- 底部状态栏：显示当前类型、字符数，以及错误位置或成功提示。

左右分栏比例默认 60% / 40%，允许用户拖动分隔条调整。

### 3.2 主题

- 提供“跟随系统 / 明亮 / 深色”三个主题选项，默认跟随 Windows 系统主题。
- 用户手动选择明亮或深色后，覆盖系统设置；选择“跟随系统”后立即恢复系统联动。
- 主题选择仅保存为本机偏好设置，不保存或上传编辑内容。

### 3.3 左侧编辑器

- 等宽字体、行号、自动换行开关和基础语法高亮。
- `格式化`：JSON 使用 2 个空格缩进；XML 保留结构并统一缩进。
- `压缩`：移除 JSON/XML 中不影响语义的空白。
- 解析成功后立即刷新右侧树；解析失败则保留左侧原文并显示错误行、列与原因。
- `复制` 将当前编辑器内容写入系统剪贴板；`清空` 清空编辑器和树。

### 3.4 右侧可编辑树

- 使用紧凑箭头（`▶` / `▼`）展开或收起节点；不额外引入 `+/-` 控件。
- 初始状态：根节点展开以显示第一层字段；第一层中包含对象或数组的节点默认收起。这与给定原型截图一致。
- JSON：可编辑对象键名、原始值（字符串、数字、布尔值、`null`）和数组元素的原始值；对象与数组节点显示子项数量。
- XML：可编辑元素名、属性名、属性值和文本节点内容；元素节点可折叠。
- 每次树内提交编辑（Enter 或失焦）后，更新内存结构并立即重新序列化到左侧编辑器；如果修改无效，保留原值并显示就近错误提示。
- 首版不提供在树内新增、删除、拖拽排序节点；可通过左侧编辑器完成结构性修改。

### 3.5 查找

顶部提供一个共享查找框、上一个/下一个匹配按钮和关闭按钮。

- 左侧：定位并高亮编辑器中的文本匹配项，显示当前序号与总数。
- 右侧：高亮匹配节点的名称、属性或值；若命中位于收起分支，自动展开其父级路径。
- 清除查询后恢复用户原有的树展开状态。

## 4. 数据同步与错误处理

应用以“已成功解析的 JSON/XML 数据结构”为唯一同步媒介，而非让左右两侧直接互相替换文本。

```text
左侧编辑文本
  └─ 解析成功 → 规范化数据结构 → 渲染右侧树

右侧树提交编辑
  └─ 更新数据结构 → 序列化 → 替换左侧编辑文本
```

- 左侧文本处于无效状态时，不覆盖用户输入；右侧继续显示上一次成功解析出的树，并在状态栏提示“当前文本无效”。
- 在该状态下通过右侧成功提交编辑，会以有效的序列化结果恢复左侧内容。
- 同步过程应带来源标记，避免左侧更新触发树更新后再次循环写回。

## 5. 最小模块划分

| 模块 | 职责 |
| --- | --- |
| `EditorPane` | 左侧文本编辑、行号、高亮、格式化和压缩入口 |
| `TreePane` | JSON/XML 树渲染、展开状态、树内编辑 |
| `documentState` | 当前类型、原文、已解析结构、错误和同步来源 |
| `formatters` | JSON/XML 的解析、格式化、压缩和序列化 |
| `SearchBar` | 查询、前后跳转、编辑器定位与树节点高亮 |
| `StatusBar` | 类型、字符数、成功和错误提示 |
| `themeState` | 系统主题监听与用户主题偏好持久化 |

不建立插件体系、格式化器注册中心或复杂命令框架；JSON/XML 的两个处理器直接放在 `formatters` 中即可。

## 6. 验收标准

1. 粘贴有效 JSON 或 XML 后，点击格式化，左侧文本被原地格式化，右侧树随即正确显示。
2. 点击压缩后，左侧文本变为单行或最少空白形式，右侧结构语义不变。
3. 树默认显示根及第一层字段；展开、收起状态可独立操作。
4. 编辑 JSON 键名或原始值，以及 XML 元素/属性/文本后，左侧内容立即更新且仍可重新解析。
5. 左侧输入无效 JSON/XML 时，显示准确错误位置，且不丢失用户输入或上一次有效树。
6. 使用查找可在左侧跳转匹配文本，并在右侧定位、高亮对应节点及其路径。
7. 完整流程在断网环境下可用，用户文本不经网络发送。
8. FormatX 默认跟随 Windows 明暗主题；用户可切换到明亮或深色，并在重启后保持该偏好。

## 7. 非目标

- YAML、SQL、CSV 或其他格式支持
- 多文档标签页、最近文件、磁盘自动保存
- 树内批量重排、拖放、节点新增/删除
- 在线账号、同步、分析或遥测
- 快捷键配置页

这些能力在用户提出明确需求后再增加；首版以快速、可靠地处理 JSON/XML 为唯一重点。
